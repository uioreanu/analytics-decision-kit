# Analytics Decision Kit

Open-source analytics starter kit for customer KPIs, revenue concentration, segmentation, visual profiling, and executive summaries.

The project is built notebook-first, but the main examples also run as plain Python scripts. Plotly outputs are generated as standalone HTML dashboards, so Jupyter is not required for the visual layer.

## Key features

- synthetic e-commerce order data generation
- optional public dataset loading from GitHub/raw CSV
- customer-level KPI calculation
- revenue deciles and Pareto-style customer concentration
- Gini index and concentration evolution
- simple customer segmentation
- KMeans customer segmentation
- RFM scoring and 5 x 5 visual profiling
- executive summary text generation
- Plotly HTML dashboards
- basic data quality and regression tests

## Quick start

```bash
git clone https://github.com/uioreanu/analytics-decision-kit.git
cd analytics-decision-kit

python -m venv .venv
.venv\Scripts\activate  # Windows

pip install -e ".[dev]"

python examples/01_customer_concentration.py
pytest
```

## Optional Plotly setup

Plotly is optional and does not require Jupyter.

Install Plotly only when you want to generate interactive HTML dashboards:

```bash
pip install plotly
```

Then run:

```bash
python examples/06_customer_analytics_dashboard.py
python examples/04_kmeans_customer_segmentation.py
```

## Python example

```python
from analytics_decision_kit.sample_data import create_demo_orders
from analytics_decision_kit.customer_analysis import run_customer_analysis

orders = create_demo_orders(n_customers=5000, n_orders=18000, seed=42)
results = run_customer_analysis(orders)

print(results["summary_text"])
print(results["decile_summary"])
```

## Optional public GitHub dataset

```python
from analytics_decision_kit.data_loader import (
    DEFAULT_SUPERSTORE_GITHUB_URL,
    get_transactional_data,
)

# No URL -> synthetic data
synthetic_orders = get_transactional_data()

# GitHub/raw CSV URL -> public dataset normalized to the same schema
public_orders = get_transactional_data(DEFAULT_SUPERSTORE_GITHUB_URL)
```

The output schema is the same in both cases:

```text
order_id, customer_id, order_date, revenue, category, manufacturer
```

## Data contract

The input dataframe should have:

| column | meaning |
|---|---|
| `customer_id` | customer id |
| `order_id` | order id |
| `order_date` | date of the order |
| `revenue` | transaction revenue |
| `category` | optional product category |
| `manufacturer` | optional manufacturer / product label |

## Main examples

| script | purpose |
|---|---|
| `examples/01_customer_concentration.py` | customer KPIs, deciles, Pareto summary |
| `examples/03_public_github_dataset.py` | load and analyze public dataset |
| `examples/04_kmeans_customer_segmentation.py` | KMeans segmentation plus Plotly dashboard |
| `examples/05_plotly_visuals_public_dataset.py` | individual Plotly HTML charts |
| `examples/06_customer_analytics_dashboard.py` | combined customer analytics dashboard |

## Public / safe by design

This project uses synthetic or publicly available data only.

The examples are generic and are not based on employer data, internal customer logic, private dashboards, or confidential business rules.

The default public dataset example uses a publicly available Sample Superstore-style CSV.
