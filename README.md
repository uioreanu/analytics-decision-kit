# Analytics Decision Kit

Analytics Decision Kit is a small, notebook-first Python package for customer analytics on transactional data. It provides synthetic data generation, public CSV normalization, customer-level KPI aggregation, customer segmentation, brand-level PCA, and standalone Plotly HTML dashboards.

The repository is intentionally generic and uses synthetic or public data only.

## What is in the kit

| Module | Purpose |
|---|---|
| `src/analytics_decision_kit/sample_data.py` | Generates synthetic transactional order data with `order_id`, `customer_id`, `order_date`, `revenue`, `category`, and `brand`. |
| `src/analytics_decision_kit/data_loader.py` | Loads synthetic data by default or normalizes supported public CSV schemas to the package contract. |
| `src/analytics_decision_kit/customer_analysis.py` | Builds customer-level KPIs, revenue deciles, Pareto summaries, order-ladder summaries, simple segments, and executive summary text. |
| `src/analytics_decision_kit/kmeans_segmentation.py` | Runs KMeans segmentation on customer-level metrics and profiles the resulting clusters. |
| `src/analytics_decision_kit/brand_pca.py` | Aggregates to brand-level metrics, adds derived KMeans cluster-share features, runs PCA, and creates a Plotly biplot dashboard. |
| `src/analytics_decision_kit/plotly_dashboards.py` | Builds the customer analytics dashboard and the KMeans segmentation dashboard as standalone HTML. |
| `src/analytics_decision_kit/cli.py` | Provides command-line entry points for the main demos and dashboards. |

## Core outputs

The standard transactional schema is:

`order_id`, `customer_id`, `order_date`, `revenue`, `category`, `brand`

All loaders and examples work toward that contract. Public datasets are normalized into the same shape where possible.

### Customer analysis outputs

`run_customer_analysis()` returns:

- `customer_metrics`
- `decile_summary`
- `pareto_summary`
- `order_ladder_summary`
- `summary_text`

### KMeans segmentation outputs

`run_kmeans_segmentation()` returns a result object with:

- customer records with assigned cluster labels
- cluster profiles
- cluster feature means
- diagnostics across candidate `k`
- selected `k`

### Brand PCA outputs

`run_brand_pca()` returns a result object with:

- `brand_metrics`
- `brand_scores` with `pc1` and `pc2`
- `loadings`
- `explained_variance`
- the fitted PCA model
- the customer metrics and KMeans result used to derive the brand features

The brand table includes both base KPIs and derived features. Derived KMeans share columns are prefixed with `derived_` so they remain distinguishable from the core brand metrics.

Contains:

```bash

```

### KMeans segmentation dashboard

[Open KMeans segmentation dashboard](docs/plotly/kmeans_customer_segmentation_dashboard.html)

Contains:

- customer share vs revenue share by cluster
- customer map by orders and revenue
- average revenue per customer by cluster

Generated by:

```bash
python examples/04_kmeans_customer_segmentation.py
```

### Brand PCA dashboard

[Open brand PCA dashboard](docs/plotly/brand_pca_dashboard.html)

Contains a Plotly biplot for brand-level PCA with:

- brand points positioned in the first two principal components
- feature loading vectors drawn as arrows
- interactive hover text for brand KPIs
- derived KMeans cluster-share features included in the PCA input table

Generated by:

```bash
python examples/07_brand_pca.py
```

## Installation

Core package:

```bash
pip install -e .
```

For development and Plotly dashboards:

```bash
pip install -e ".[dev,viz]"
```

For the full local toolchain, including notebook dependencies:

```bash
pip install -e ".[all]"
```

## Quick start

```bash
git clone https://github.com/uioreanu/analytics-decision-kit.git
cd analytics-decision-kit

python -m venv .venv
.venv\Scripts\activate  # Windows

pip install -e ".[dev,viz]"
python examples/01_customer_concentration.py
python examples/07_brand_pca.py
pytest
```

## Example workflows

### Customer concentration

```bash
python examples/01_customer_concentration.py
```

This runs the customer KPI pipeline, revenue deciles, Pareto summary, and the order-ladder summary.

### Brand PCA

```bash
python examples/07_brand_pca.py
```

This script:

1. creates synthetic transactional data,
2. builds customer metrics,
3. runs KMeans segmentation,
4. aggregates brand-level metrics,
5. fits PCA on the brand feature matrix,
6. writes [docs/plotly/brand_pca_dashboard.html](docs/plotly/brand_pca_dashboard.html).

### Command-line entry points

If you install the package in editable mode, the repository exposes console commands through `pyproject.toml`:

- `adk-demo`
- `adk-dashboard`
- `adk-kmeans`
- `adk-brand-pca`

## Data contract

The package expects transactional data with the following fields:

| Column | Type | Meaning |
|---|---|---|
| `order_id` | string | unique transaction identifier |
| `customer_id` | string | unique customer identifier |
| `order_date` | datetime | transaction date |
| `revenue` | numeric | transaction revenue |
| `category` | string | product category or generic grouping |
| `brand` | string | product brand or brand-like token |

Public CSVs are normalized into this schema when possible. The default synthetic generator already emits this contract.

## Development notes

- The package is designed to work with plain Python scripts as well as notebook cells.
- Plotly outputs are standalone HTML files; Jupyter is optional for dashboard generation.
- Tests live under [tests/](tests/).
- Generated HTML outputs live under [docs/plotly/](docs/plotly/).

## Public and safe by design

The repository uses synthetic data and public datasets only. It does not depend on employer-specific logic, private dashboards, or confidential business rules.
