# KPI definitions

## Customer

One unique customer id.

Default column:

```text
customer_id
```

## Revenue

Revenue value per order.

Can be gross or net revenue. 

## Orders

Unique count of order ids per customer.

```text
nunique(order_id)
```

## Average order value

```text
revenue / orders
```

## Repeat customer

```text
orders >= 2
```

## Revenue decile

Customers sorted by revenue descending and split into 10 groups.

- decile 1 = highest revenue customers
- decile 10 = lowest revenue customers

## Top customer concentration

Revenue share generated by top x% customers.

Example:

```text
top 10% customers generate 48% of revenue
```

## Segment logic

Current simple demo logic:

| segment | rule |
|---|---|
| High Value | revenue >= 90th percentile |
| Mid Value Repeat | orders >= 2 and below high value line |
| One-Time Buyer | orders = 1 |
| Low Activity | fallback |
